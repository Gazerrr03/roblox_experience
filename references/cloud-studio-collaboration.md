# Roblox 开发协作概念（给新手的简单说明）

## 两个工具，各干各的

### Roblox Studio 云端实例 = 大家一起用的"白板"
- 一个人搭好项目，分享链接
- 其他人点进来一起编辑、玩耍、调试
- 场景、资产、关卡都在这里

### Git = 代码的"网盘同步"
- 程序在这里写代码
- 提交、Review、合并
- 新代码通过 `rojo serve` 推到 Studio

## 工作方式

```
1. 主创在 Studio 建好项目，分享链接
2. 所有人进这个云端项目玩
3. 程序本地写完代码 → git push
4. rojo serve 自动把代码送进 Studio
5. 大家直接在 Studio 看到新代码的效果
```

## 简单理解

- **想改场景/关卡？** → 进 Studio 改
- **想改玩法逻辑？** → 程序员 git push，rojo serve 自动同步

不需要懂太多流程，进 Studio 就能参与协作。
