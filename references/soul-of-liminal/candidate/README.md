# Candidate Layer

这里仍然是 `Soul of Liminal` 的活跃思考层，但现在承担三种职责：

1. 当前 slice 的根本文档
2. 来源、样例与论据文档
3. 尚未正式化的未来增强方向

## 当前最重要的 candidate 文档

- `deadline-slice.md`
  - 当前 deadline slice 的根本 framing
- `deadline-sprint.md`
  - 当前 slice 的 execution companion

这两篇的优先级高于其他历史 candidate 文档。
本轮新增 formal 文档不能推翻它们的核心 promise，只能把它们落成更清晰的当前确认版规则。

## 当前新增 candidate 文档

- `crew-dynamics-and-information-asymmetry.md`
  - 角色分工、私有信息、口头传播与未来误导层
- `reveal-ladder.md`
  - 身份真相、Pages 真相与上位系统的揭示阶梯

它们暂时保留在 candidate，因为：
- 它们对长期辨识度很重要
- 但当前 slice 还不应该被它们反客为主
- 它们还需要继续验证，不适合压成 formal 硬规则

## 旧 candidate 文档如何使用

- `project-philosophy.md` / `game-pillars.md`
  - 价值主轴来源
- `systems/*`
  - 局部系统方向、样例与来源碎片
- `clues/*`

- `cards/*`
  - 高密度原则卡，便于快速引用
