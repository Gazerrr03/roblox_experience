这一轮里，“formal” 的含义不是“永远不会改”，而是：

- 已经足够稳定
- 可以直接约束当前 deadline slice 的设计、写作、内容生产与复盘

## Formal 的使用规则

- formal 用来回答“当前到底怎么做”
- formal 不负责保存全部来源讨论；来源仍保留在 candidate
- formal 的 `Current Decision` 必须足够清楚，能直接限制后续写作与实现
- 如果未来 slice 根本变化，先改 `candidate/deadline-slice.md`，再回收 formal
