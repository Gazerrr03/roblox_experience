# [Maze][UI] 小地图 + 迷雾探索系统

## 问题 / 需求背景

maze 关卡改为人工设计后，玩家需要一个帮助辨认整体布局的工具——小地图。同时为了营造探索感，采用"迷雾系统"：玩家只能看到自己走过的房间，未探索区域保持纯黑遮蔽。

## 需求描述

### 迷雾系统
- **触发条件**：玩家位置（XZ 平面投影）进入某房间投影区域即算进入新房间，基于六边形面积检测（非门触发）
- **检测范围**：仅 `RoomType/*` 标签的实例参与面积检测；其他类型前缀（`SpatialType/`、`SceneryType/` 等）由你自定义，不参与检测
- **解锁范围**：每个人独立探索，解锁永久保留
- **道具共享**：通讯器（communicator）道具——握持者的小地图视野与全队共享（持续共享，非 ping）
- **视觉表现**：已探索房间保持纯黑（无轮廓）；门不可见；玩家点可见

### 小地图
- **渲染方案**：ViewportFrame + 正交俯视相机
- **更新时机**：基于 `MazeWorldScanner` 扫描结果初始化，随探索状态更新
- **载体**：Roblox ScreenGui，客户端本地渲染（服务端只下发探索状态）

### 技术要求
- `RoomType/*` 硬编码为迷雾检测类型；其他 Tag 前缀由你定义，扫描时收集但不参与面积检测
- 迷雾状态存储在客户端（`PlayerStateService` 或独立模块）
- 服务端只负责通讯器共享逻辑的广播
- 小地图 VPF 与 3D 世界渲染完全解耦

## 验收标准
- [ ] 玩家进入某房间投影区域后，对应房间标记为已探索
- [ ] 每个人独立探索，解锁状态互不影响
- [ ] 持有通讯器时，小地图视野与全队同步
- [ ] 已探索房间保持纯黑；门在未探索时不可见
- [ ] 切换至 FallbackReadable 兼容性 Technology 时小地图正常运作
